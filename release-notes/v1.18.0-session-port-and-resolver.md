# Session porting (codex ↔ claude) + resolver integration

## What

`aichat port <query>` converts a session from one coding agent into a
**native, resumable** session of the other agent:

- **Codex → Claude**: a codex rollout becomes a Claude Code session JSONL
  under `~/.claude/projects/<project>/`, resumable with
  `claude --resume <new-id>`.
- **Claude → Codex**: a Claude session becomes a codex rollout under
  `~/.codex/sessions/YYYY/MM/DD/` (UUIDv7 id, `history.jsonl`
  registration), resumable with `codex resume <new-id>`.

The query can be a full ID, partial ID (prefix, middle, or suffix), a
session **name**, a codex rollout timestamp fragment, or a file path.
Lookup is global across all projects in both agent homes.

## How conversion works

- User/assistant turns carry over as real transcript history.
- Tool calls and results are flattened to labeled text (agents have
  incompatible tool schemas); tool args/results are truncated past a cap,
  normal message text is preserved verbatim.
- Agent-internal noise is dropped: encrypted reasoning, injected
  instruction wrappers, thinking blocks, sidechains, command wrappers,
  inter-agent notification envelopes.
- Streaming conversion with atomic writes; `continue_metadata` lineage on
  the first line so `aichat lineage` tracks the cross-agent parent.

## Resolver changes

`aichat port` resolves via `resolve_session.resolve()` (shared with
`aichat resolve`), extended with new tiers:

1. exact id → 2. exact name → 3. id prefix → 4. id substring →
5. filename substring → 6. name substring

Filename fragments rank above name substrings so a structural match
(one rollout file) is not drowned by sessions whose free-text names quote
the same timestamp. Codex explicit thread names from
`~/.codex/session_index.jsonl` now resolve (the state DB's `title` is
just the auto-captured first message).

## Testing

- 388 port/resolver tests (no mocks, tmp homes); full suite green apart
  from 4 pre-existing environment-dependent tmux tests.
- Real end-to-end verification in both directions: ported sessions were
  resumed with `claude --resume` / `codex exec resume` and correctly
  answered content questions about the original conversation.
- Built via codex (GPT) adversarial review loops: ~90 blocking findings
  across the three phases (implement codex→claude, implement
  claude→codex, merge main + resolver integration) were fixed at root
  cause before landing.
