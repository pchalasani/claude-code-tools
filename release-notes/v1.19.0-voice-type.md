# v1.19.0 — voice-type: local voice dictation CLI

New tool: `voice-type`, fully on-device speech-to-text that types wherever your
cursor is — editor, browser, terminal, or a coding agent's prompt box.

## Highlights

- Local transcription via Moonshine streaming models (medium-streaming default;
  no cloud, no accounts) with built-in voice activity detection. Utterances are
  typed atomically on pause, so each one is a single undo unit.
- Three activation modes:

  - `toggle` — start paused; a global hotkey (default `<ctrl>+;`) starts/stops
  - `vad` — hands-free; types every utterance
  - `wake` — passive until a configurable wake word (default "claude");
    re-arms on a stop phrase ("stop listening") or silence timeout

- Spoken submit phrases: saying "over", "go", or "submit" as an entire
  utterance presses Enter in the focused app — dictate a prompt to a coding
  agent and submit it without touching the keyboard. Mid-sentence occurrences
  are typed normally.
- macOS event-tap hotkey suppression: the toggle chord is swallowed at the
  CGEvent level, so binding e.g. `Ctrl+;` never types a stray ";" into the
  focused app. Auto-repeats and the key-up are suppressed too.
- TOML config at `~/.config/voice-type/config.toml` (`voice-type init` writes a
  commented sample); every setting is also a CLI flag.
- Two local engines, selectable via `engine` / `--engine`:

  - `moonshine` (default) — Moonshine streaming models (`voice` extra)
  - `parakeet` — NVIDIA Parakeet-TDT 0.6b v3 via sherpa-onnx + Silero VAD
    (`voice-parakeet` extra; ~490 MB one-time download; cleaner transcripts,
    natively drops filler words)

- `strip_fillers` (default on): removes standalone "uh"/"um"-style words from
  typed text on any engine.
- Wake-mode idle timeout now defers while speech is in progress (engines
  report VAD/partial-text activity), so a long dictation can't re-arm the
  wake word mid-sentence.

## Install

```bash
uv tool install "claude-code-tools[voice]"
# or zero-install:
uvx --from "claude-code-tools[voice]" voice-type
```

Dependencies (`moonshine-voice`, `pynput`) live behind the new `voice` extra,
so the base package stays lean.

## Implementation notes

- New subpackage `claude_code_tools/voice_type/` (~700 lines incl. tests):
  `cli.py`, `config.py`, `logic.py` (pure wake/stop/submit phrase matching),
  `app.py` (activation state machine + Moonshine wiring), `inject.py`
  (pynput keystrokes), `hotkey.py` (macOS suppressing hotkey with
  keyboard-layout keycode probe; non-mac falls back to `GlobalHotKeys`).
- 20 unit tests covering config round-trip/validation, phrase matching with
  word boundaries, and hotkey parsing.
- Docs: `docs-site/src/content/docs/tools/voice-type.mdx` + sidebar entry.
