# v1.19.0 — voice-type: local voice dictation CLI

New tool: `voice-type`, fully on-device speech-to-text that types wherever your
cursor is — editor, browser, terminal, or a coding agent's prompt box.

## Highlights

- Local transcription via Moonshine streaming models (medium-streaming default;
  no cloud, no accounts) with built-in voice activity detection. Utterances are
  typed atomically on pause, so each one is a single undo unit.
- Three activation modes:

  - `toggle` — start paused; a global hotkey (default `<ctrl>+;`) starts/stops
  - `vad` — hands-free; types every utterance
  - `wake` — passive until a configurable wake word (default "claude");
    re-arms on a stop phrase ("stop listening") or silence timeout

- Spoken submit phrases: saying "over", "go", or "submit" as an entire
  utterance presses Enter in the focused app — dictate a prompt to a coding
  agent and submit it without touching the keyboard. Mid-sentence occurrences
  are typed normally.
- macOS event-tap hotkey suppression: the toggle chord is swallowed at the
  CGEvent level, so binding e.g. `Ctrl+;` never types a stray ";" into the
  focused app. Auto-repeats and the key-up are suppressed too.
- TOML config at `~/.config/voice-type/config.toml` (`voice-type init` writes a
  commented sample); CLI flags cover the common settings (mode, engine,
  hotkey, wake word, ...) — the config file covers everything.
- Two local engines, selectable via `engine` / `--engine`:

  - `moonshine` (default) — Moonshine streaming models (`voice` extra)
  - `parakeet` — NVIDIA Parakeet-TDT 0.6b v3 via sherpa-onnx + Silero VAD
    (`voice-parakeet` extra; ~490 MB one-time download; cleaner transcripts,
    natively drops filler words)

- `strip_fillers` (default on): removes standalone "uh"/"um"-style words from
  typed text on any engine.
- Automatic gain control in the Parakeet engine: quiet dynamic mics (e.g.
  Shure MV7) deliver peaks far below Silero VAD's detection range; an
  instant-attack / fast-release AGC brings speech to VAD-friendly levels.
- `segmentation = "hold"` (parakeet + toggle mode): record from toggle-on to
  toggle-off and transcribe the whole take as one segment — full context, no
  mid-sentence VAD chopping, typed in one go (Hex-style push-to-talk feel).
- `parakeet_model = "v3-int8" | "v2-fp16"`: choose the quantized multilingual
  build or the higher-precision English fp16 build (~1.1 GB).
- Third engine `parakeet-mlx`: Parakeet-TDT v3 at fp16 on the Apple GPU via
  MLX — ~40x realtime with full-precision accuracy (the recommended engine;
  Apple Silicon only, `voice-mlx` extra).
- Recording indicator: a floating waveform pill (click-through,
  non-activating) appears ONLY while recording — red waves as you speak.
- 3+ word stutters collapse ("I I I think" → "I think"); two repeats are
  left alone.
- Configurable start/stop sounds (`sound_start`/`sound_stop`: system sound
  names or file paths).
- Transcript rescue: optional `copy_to_clipboard`, and `paste_hotkey` re-types
  the last session's transcript at the cursor (wrong-window recovery).
- `cancel_hotkey` (default Esc, intercepted only while recording) discards a
  recording in progress.
- `voice-type hotkey` records a pressed chord and prints the config line;
  bracket-less hotkey notation (`ctrl+;`) accepted everywhere.
- Robustness from live testing: AGC for quiet dynamic mics with a noise-safe
  gain cap, stuck-VAD self-recovery, VAD reset after every flush (sherpa
  circular-buffer corruption), toggle debounce for macro keyboards, wake-word
  aliases for transcriber misspellings, and no silent drops — every utterance
  logs its outcome.
- Docs: dedicated Starlight section (Overview + Configuration & CLI) with a
  new card on the README and landing-page grids.
- Wake-mode idle timeout now defers while speech is in progress (engines
  report VAD/partial-text activity), so a long dictation can't re-arm the
  wake word mid-sentence.

## Install

```bash
uv tool install "claude-code-tools[voice]"
# or zero-install:
uvx --from "claude-code-tools[voice]" voice-type
```

Dependencies (`moonshine-voice`, `pynput`) live behind the new `voice` extra,
so the base package stays lean.

## Implementation notes

- New subpackage `claude_code_tools/voice_type/`: `cli.py`, `config.py`,
  `logic.py` (pure wake/stop/submit phrase matching), `app.py` (activation
  state machine), `engines.py` / `engine_parakeet.py` / `engine_parakeet_mlx.py`
  (transcription backends), `inject.py` (pynput keystrokes), `overlay.py`
  (waveform pill), and `hotkey.py` (macOS suppressing hotkey with
  keyboard-layout keycode probe; non-mac falls back to `GlobalHotKeys`).
- Unit tests (`tests/test_voice_type.py`, `tests/test_voice_type_engines.py`)
  covering config round-trip/validation, phrase matching, hotkey parsing, the
  activation state machine, and the engines' capture/recovery paths.
- Docs: `docs-site/src/content/docs/tools/voice-type/` (Overview +
  Configuration & CLI) + sidebar entries.
